## Phase Details

### Phase 31: Q1B Branch (Inferno contra-fobico)

**Goal:** Visitors with q1=B && q2=B pattern (destemido profile) hear Q1B "A Porta no Fundo" branch, testing courage against emptiness.

**Depends on:** Nothing (first phase of v6.0)

**Requirements:** BR-01, POL-02

**Success Criteria** (what must be TRUE):
1. Visitor who exits room (q1=B) AND keeps looking at the thing (q2=B) triggers Q1B branch after Q2_RESPOSTA_B
2. NLU correctly classifies "atravessar/voltar" choice pair with keywords from QUESTION_META[9]
3. Guard `shouldBranchQ1B` does not trigger for any other choice patterns (Q1=A, Q2=A, or mixed)
4. Branch adds ~30-45s to experience timing when triggered
5. All v4.0 tests continue passing (ChoiceMap extension is backward compatible)

### Phase 32: Q5B Branch (Paraiso gap closure)

**Goal:** Visitors with q4=A && q5=A pattern (full memory + carrying question) hear Q5B "O Que Já Não Cabe" branch, closing Paraiso branching gap.

**Depends on:** Phase 31

**Requirements:** BR-02

**Success Criteria** (what must be TRUE):
1. Visitor who remembers everything (q4=A) AND carries question without answer (q5=A) triggers Q5B branch after Q5_RESPOSTA_A
2. NLU correctly classifies "fundir/ordenar" choice pair with keywords from QUESTION_META[10]
3. Branch triggers only in Paraiso (after Q5_RESPOSTA_A), not in Inferno or Purgatorio
4. Q5B does not interfere with other Paraiso paths (visitors who chose q4=B or q5=B skip Q5B)
5. Paraiso now has branching parity with Inferno and Purgatorio

### Phase 33: Q6B + ESPELHO_SILENCIOSO

**Goal:** Visitors with q5=B && q6=A pattern (dissolved question + wants reading) hear Q6B "O Espelho Extra" branch, leading to rare DEVOLUCAO_ESPELHO_SILENCIOSO archetype instead of normal devolution.

**Depends on:** Phase 32

**Requirements:** BR-03, AR-01

**Success Criteria** (what must be TRUE):
1. Visitor who dissolved their question (q5=B) BUT asked to hear Oracle (q6=A) triggers Q6B branch before DEVOLUCAO
2. Q6B choice "O que o Oraculo viu" (q6b=A) routes to normal DEVOLUCAO with 8 existing archetypes
3. Q6B choice "O que eu teria visto" (q6b=B) routes to DEVOLUCAO_ESPELHO_SILENCIOSO state instead of normal devolution
4. Guard `isEspelhoSilencioso` has highest priority in DEVOLUCAO `always` transitions (checks before 8 existing archetypes)
5. ESPELHO_SILENCIOSO devolution returns open structure (~22-28s, ends with question) instead of closed diagnosis
6. All 8 existing archetypes still trigger correctly for visitors who don't reach Q6B

### Phase 34: Detectable Archetypes

**Goal:** Add 2 new detectable archetypes (CONTRA_FOBICO, PORTADOR) leveraging new branch choice data to deepen pattern reading.

**Depends on:** Phase 33

**Requirements:** AR-02, AR-03

**Success Criteria** (what must be TRUE):
1. Visitor who traversed the void (q1=B && q2=B && q1b=A) triggers DEVOLUCAO_CONTRA_FOBICO archetype with script "você procura medos do tamanho da sua coragem"
2. Visitor who fused memory + question (q4=A && q5=A && q5b=A) triggers DEVOLUCAO_PORTADOR archetype with script "você não carrega — você se transforma no peso"
3. Guard priority order in DEVOLUCAO `always` transitions: ESPELHO_SILENCIOSO → CONTRA_FOBICO → PORTADOR → 8 existing archetypes
4. All 8 existing archetypes still match correctly when new guards don't trigger (backward compatibility preserved)
5. Guards `isContraFobico` and `isPortador` defined in `src/machines/guards/patternMatching.ts` follow existing pattern matching conventions

### Phase 35: Timing Mitigation + Browser UAT

**Goal:** Validate timing across all 96 flow permutations, mitigate max-path overflow if needed, and verify experience in browser across representative paths.

**Depends on:** Phase 34

**Requirements:** POL-01, POL-03, UAT-01

**Success Criteria** (what must be TRUE):
1. `scripts/validate-timing.ts` updated to validate all 96 path permutations (including new branches Q1B, Q5B, Q6B)
2. Max-path documented for worst-case flow (Q1B + Q4B + Q5B + Q6B all triggered)
3. If max-path exceeds 7:30, mitigation applied (trim base SETUPs for Q1/Q3/Q5) OR overflow accepted and documented in PROJECT.md with ~1-3% visitor estimate
4. Browser UAT validates 3 representative paths: (a) no new branches triggered, (b) all branches Q1B+Q4B+Q5B triggered, (c) Q6B → DEVOLUCAO_ESPELHO_SILENCIOSO
5. `public/roteiro.html` updated with 3 new branches in Mermaid flowchart + narrative text, maintaining parity with `src/data/script.ts`
6. CLAUDE.md updated with new counts (script entries: 61 → 85, states: ~54 → ~78)
