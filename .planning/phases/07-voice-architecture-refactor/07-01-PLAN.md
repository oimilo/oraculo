    **Step 2: Refactor useVoiceChoice to useReducer FSM** (`src/hooks/useVoiceChoice.ts`)

    Replace the 5 useState declarations (isProcessing, choiceResult, needsFallback, attemptCount, error) with a single useReducer implementing an explicit finite state machine.

    Define the types:
    ```typescript
    export type VoiceLifecyclePhase = 'idle' | 'listening' | 'processing' | 'decided' | 'fallback';

    export type VoiceLifecycle =
      | { phase: 'idle' }
      | { phase: 'listening'; startedAt: number }
      | { phase: 'processing'; blob: Blob; attempt: number }
      | { phase: 'decided'; result: VoiceChoiceResult }
      | { phase: 'fallback'; attempt: number };

    export type VoiceEvent =
      | { type: 'START_LISTENING' }
      | { type: 'AUDIO_READY'; blob: Blob }
      | { type: 'CLASSIFICATION_COMPLETE'; result: VoiceChoiceResult }
      | { type: 'NEED_FALLBACK'; attempt: number }
      | { type: 'RESET' };
    ```

    Export the reducer function `voiceLifecycleReducer` so it can be unit tested directly:
    ```typescript
    export function voiceLifecycleReducer(state: VoiceLifecycle, event: VoiceEvent): VoiceLifecycle
    ```

    Transition table:
    - idle + START_LISTENING -> { phase: 'listening', startedAt: Date.now() }
    - listening + AUDIO_READY -> { phase: 'processing', blob, attempt: (prev attempt from fallback or 1) }
    - processing + CLASSIFICATION_COMPLETE -> { phase: 'decided', result }
    - processing + NEED_FALLBACK -> { phase: 'fallback', attempt }
    - fallback + START_LISTENING -> { phase: 'listening', startedAt: Date.now() }
    - decided + RESET -> { phase: 'idle' }
    - Any other combination -> return state unchanged (no-op)

    The hook body:
    - `const [lifecycle, dispatch] = useReducer(voiceLifecycleReducer, { phase: 'idle' })`
    - Move the audio processing logic (STT + NLU) into a useEffect that triggers on lifecycle.phase === 'processing'
    - When audioBlob arrives from useMicrophone AND lifecycle.phase === 'listening', dispatch AUDIO_READY
    - When active=true, dispatch START_LISTENING; when active=false, dispatch RESET
    - Derive backward-compatible return values from lifecycle:
      - `isListening = lifecycle.phase === 'listening'`
      - `isProcessing = lifecycle.phase === 'processing'`
      - `needsFallback = lifecycle.phase === 'fallback'`
      - `choiceResult = lifecycle.phase === 'decided' ? lifecycle.result : null`
      - `attemptCount` = derive from lifecycle (processing.attempt or fallback.attempt or 0)

    Keep the same `UseVoiceChoiceReturn` interface for backward compatibility, BUT also expose `lifecycle` and `dispatch` for advanced consumers:
    ```typescript
    export interface UseVoiceChoiceReturn {
      // Backward compatible
      isListening: boolean;
      isProcessing: boolean;
      choiceResult: VoiceChoiceResult | null;
      needsFallback: boolean;
      attemptCount: number;
      error: string | null;
      startListening: () => Promise<void>;
      stopListening: () => void;
      reset: () => void;
      // New: explicit lifecycle
      lifecycle: VoiceLifecycle;
      dispatch: React.Dispatch<VoiceEvent>;
    }
    ```

    Keep `ChoiceConfig` and `VoiceChoiceResult` interfaces exactly as they are — no changes.
