  <action>
Create scripts/validate-timing.ts — a standalone script that calculates experience duration.

The script must:

1. Import SCRIPT from '../src/data/script.ts'
2. Define a function `calculateSectionDuration(segments: SpeechSegment[]): number` that:
   - For each segment, estimates speech duration as `text.length / 13` seconds (PT-BR speech rate ~13 chars/sec)
   - Adds `pauseAfter / 1000` seconds for each segment that has pauseAfter
   - Returns total seconds for the section

3. Define MAX_PATH_SECTIONS as an ordered array of SCRIPT keys representing the longest possible path through the experience:
   ```
   APRESENTACAO
   INFERNO_INTRO
   INFERNO_Q1_SETUP, INFERNO_Q1_PERGUNTA, INFERNO_Q1_RESPOSTA_A (or B, use longer one)
   INFERNO_Q2_SETUP, INFERNO_Q2_PERGUNTA, INFERNO_Q2_RESPOSTA_A (or B, use longer one)
   PURGATORIO_INTRO
   PURGATORIO_Q3_SETUP, PURGATORIO_Q3_PERGUNTA, PURGATORIO_Q3_RESPOSTA_A (or B)
   PURGATORIO_Q4_SETUP, PURGATORIO_Q4_PERGUNTA, PURGATORIO_Q4_RESPOSTA_A (or B)
   PARAISO_INTRO
   PARAISO_Q5_SETUP, PARAISO_Q5_PERGUNTA, PARAISO_Q5_RESPOSTA_A (or B)
   PARAISO_Q6_SETUP, PARAISO_Q6_PERGUNTA, PARAISO_Q6_RESPOSTA_A (or B)
   longest DEVOLUCAO_* entry
   ENCERRAMENTO
   ```

4. For each question pair (RESPOSTA_A vs RESPOSTA_B), automatically pick the LONGER one for max-path.

5. Add 4 seconds per question for listener response time (6 questions = 24 seconds total).

6. Print a detailed breakdown:
   ```
   === V4.0 TIMING VALIDATION ===

   Section                    | Segs | Chars | Duration
   ---------------------------------------------------
   APRESENTACAO               |    4 |   XXX |   XX.Xs
   INFERNO_INTRO              |    1 |   XXX |    X.Xs
   ...
   ---------------------------------------------------
   SUBTOTAL (narration)       |   XX |  XXXX |  XXX.Xs
   Listener response time     |      |       |   24.0s
   TOTAL MAX-PATH             |      |       |  XXX.Xs = X.X min

   TARGET: 5-7 minutes (300-420 seconds)
   STATUS: PASS / FAIL (X.X min)
   ```

7. Also compute and print min-path (using shorter respostas) and average-path.

8. Exit with code 0 if max-path is within 300-420 seconds, exit code 1 otherwise.

Use `tsx` to run (already in devDependencies). The script should be runnable as:
```
npx tsx scripts/validate-timing.ts
```

TypeScript configuration: use path alias resolution compatible with the project's tsconfig (the project uses `@/` path alias for `src/`). If path aliases cause issues, use relative imports instead: `import { SCRIPT } from '../src/data/script'`.
