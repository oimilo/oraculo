Build the XState v5 state machine using `createMachine` from 'xstate'. Follow the exact structure from Research Pattern 1 (lines 568-746 of 01-RESEARCH.md) with these specifications:

**Machine ID:** 'oracle'
**Initial state:** 'IDLE'

**State hierarchy:**
- IDLE (flat)
- APRESENTACAO (flat)
- INFERNO (nested: NARRATIVA, PERGUNTA, AGUARDANDO, TIMEOUT_REDIRECT, RESPOSTA_A, RESPOSTA_B)
- PURGATORIO_A (nested: NARRATIVA, PERGUNTA, AGUARDANDO, RESPOSTA_FICAR, RESPOSTA_EMBORA) with id: 'PURGATORIO_A'
- PURGATORIO_B (nested: NARRATIVA, PERGUNTA, AGUARDANDO, RESPOSTA_PISAR, RESPOSTA_CONTORNAR) with id: 'PURGATORIO_B'
- PARAISO with id: 'PARAISO' (flat -- NARRATIVA_DONE transitions to DEVOLUCAO)
- DEVOLUCAO (flat with `always` guards routing to variant states based on choice1 + choice2)
- DEVOLUCAO_A_FICAR, DEVOLUCAO_A_EMBORA, DEVOLUCAO_B_PISAR, DEVOLUCAO_B_CONTORNAR (flat, each -> ENCERRAMENTO on NARRATIVA_DONE)
- ENCERRAMENTO (flat -> FIM on NARRATIVA_DONE)
- FIM (after 5000ms -> IDLE with context reset)

**Context updates via assign():**
- START: assign sessionId via crypto.randomUUID(), reset choice1/choice2/fallbackCount to initial values
- CHOICE_A: assign choice1 = 'A'
- CHOICE_B: assign choice1 = 'B'
- CHOICE_FICAR: assign choice2 = 'FICAR'
- CHOICE_EMBORA: assign choice2 = 'EMBORA'
- CHOICE_PISAR: assign choice2 = 'PISAR'
- CHOICE_CONTORNAR: assign choice2 = 'CONTORNAR'
- Entry to major phases: assign currentPhase accordingly
- FIM -> IDLE transition: reset all context to INITIAL_CONTEXT values

**Timeout configurations (per PRD):**
- INFERNO.AGUARDANDO: after 15000ms -> TIMEOUT_REDIRECT (with choice1 = 'B')
- INFERNO.TIMEOUT_REDIRECT: after 2000ms -> RESPOSTA_B
- PURGATORIO_A.AGUARDANDO: after 15000ms -> RESPOSTA_FICAR (with choice2 = 'FICAR')
- PURGATORIO_B.AGUARDANDO: after 15000ms -> RESPOSTA_CONTORNAR (with choice2 = 'CONTORNAR')

**DEVOLUCAO guards (using `always` transitions):**
```typescript
always: [

  { target: 'DEVOLUCAO_A_EMBORA', guard: ({ context }) => context.choice1 === 'A' && context.choice2 === 'EMBORA' },
  { target: 'DEVOLUCAO_B_PISAR', guard: ({ context }) => context.choice1 === 'B' && context.choice2 === 'PISAR' },
  { target: 'DEVOLUCAO_B_CONTORNAR', guard: ({ context }) => context.choice1 === 'B' && context.choice2 === 'CONTORNAR' },
]
```

**Cross-state references:** Use `#oracle.PURGATORIO_A`, `#oracle.PURGATORIO_B`, `#oracle.PARAISO` for transitions that cross hierarchy boundaries.
